# 烟台文化旅游职业学院学生信息核验管理系统

本项目是面向“烟台文化旅游职业学院”的学生信息核验管理系统，用于导入标准学生库和待核对学生表，按学号自动比对学生基础信息、资助信息，并输出异常结果。

## 学校品牌信息

- 学校中文名称：烟台文化旅游职业学院
- 学校英文名称：YANTAI VOCATIONAL COLLEGE OF CULTURE AND TOURISM
- 系统名称：学生信息核验管理系统
- 副标题：校园学生基础与资助信息数字化核验平台
- 校徽图片路径：`assets/school-logo.jpg`

## Excel 导入模板字段

系统支持 `.xlsx` 和 `.xls` 文件，自动读取第一个工作表。

必填表头：

- 姓名
- 学号
- 性别
- 班级
- 联系方式
- 身份证号
- 银行卡号

可选表头：

- 年龄
- 家庭住址
- 开户银行
- 资助项目
- 资助金额
- 备注

导入时会自动去除字段前后空格，学号、联系方式、身份证号和银行卡号会按字符串处理；联系方式、身份证号会去除空格，银行卡号会去除空格和横线。

标准学生库已有数据时，导入 Excel 会提示选择导入方式：

- 合并导入：按学号判断，新增不存在的学生，跳过已存在的学号。
- 覆盖导入：二次确认后清空当前标准学生库，并用 Excel 数据替换。
- 取消导入：不修改任何数据。

待核对表导入默认覆盖，因为待核对表通常用于一次性核验。

## 核验规则

点击“开始比对”后，系统按学号比对标准学生库与待核对表：

- 学号为空：无效数据
- 学号重复：重复学号
- 待核对表中的学号在标准库中不存在：未录入学生
- 标准库中存在学号，但姓名、性别、年龄、班级、联系方式、身份证号、银行卡号、开户银行、资助项目、资助金额、家庭住址任一字段不同：信息不一致
- 标准库中存在但待核对表缺少对应学号：待核对表缺失
- 所有核心字段一致：通过

备注字段不参与自动比对。

## 功能

- 登录后进入学生信息核验后台
- 导入标准学生库 Excel
- 导入待核对表 Excel
- 下载标准学生库和待核对表 Excel 模板，减少表头填写错误
- 支持身份证号、银行卡号、开户银行、资助项目、资助金额等资助信息字段
- 支持选择参与自动比对的字段，适配不同核验场景
- 支持导入错误清单，显示具体行号、字段和错误原因
- 支持核对异常标记处理、填写处理意见和保存处理时间
- 支持脱敏显示、脱敏导出和完整导出结果
- 支持本地 JSON 备份导出和备份恢复，便于迁移浏览器或电脑
- 标准学生库支持合并导入、覆盖导入和取消导入，避免误覆盖手动新增数据
- 自动核验与异常结果展示
- 查看标准学生库、待核对表、核对结果
- 手动新增、编辑、删除学生基础与资助信息
- 搜索、班级筛选、排序
- 统计学生总数、核对通过、异常数据、重复学号
- 导出核对结果，优先导出 Excel，解析库不可用时导出 CSV
- 记录本地操作日志，并随备份文件一起导出
- 数据保存到浏览器 `localStorage`

## 应用版使用说明

本项目已支持 PWA，可在 Chrome、Edge 等现代浏览器中安装为桌面应用。

### 本地运行

直接打开 `index.html` 可以使用大部分功能。若要完整测试 PWA 安装、Service Worker 和离线缓存，建议使用本地静态服务器，例如：

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

### GitHub Pages 部署

1. 将项目推送到 GitHub 仓库。
2. 打开仓库 Settings。
3. 进入 Pages。
4. Source 选择当前分支，例如 `master`。
5. 目录选择 `/root`。
6. 保存后等待 GitHub Pages 生成访问地址。

部署后通过 HTTPS 访问页面，浏览器会自动识别 PWA 配置。

### 在 Edge 或 Chrome 中安装

1. 使用 Edge 或 Chrome 打开系统页面。
2. 地址栏右侧出现“安装”图标时点击安装。
3. 也可以点击页面中的“可安装为桌面应用”提示。
4. 安装完成后，可从桌面或开始菜单打开“学生核验系统”。

### 手机端添加到主屏幕

- Android Chrome：打开页面后，点击浏览器菜单，选择“添加到主屏幕”或“安装应用”。
- iPhone Safari：打开页面后，点击分享按钮，选择“添加到主屏幕”。

### 离线使用说明

系统会缓存基础页面资源，包括：

- `index.html`
- `styles.css`
- `app.js`
- `manifest.json`
- 校徽和应用图标

首次在线打开并加载成功后，后续可在离线状态下打开基础页面。Excel 解析库来自 CDN，若首次缓存失败，离线时 Excel 导入导出可能受影响。

## 数据保存方式

数据保存在当前浏览器的 `localStorage` 中，包括：

- `standardStudents` 标准学生库
- `checkStudents` 待核对学生表
- `verificationResults` 核对结果
- `importErrors` 导入错误清单
- `settings` 脱敏显示和核验字段设置
- `audit` 本地操作日志

清理浏览器缓存或更换浏览器可能导致本地数据丢失。

PWA 安装不会把数据上传到服务器，数据仍保存在当前浏览器或应用容器的本地存储中。建议在正式核验前后使用“导出备份”保存 JSON 备份；更换电脑或浏览器后，可通过“导入备份”恢复本地数据。

## 本轮安全与可用性优化说明

针对前端本地版的主要缺陷，系统已做以下优化：

- 数据安全：默认脱敏显示身份证号和银行卡号，导出结果时可选择脱敏导出或完整导出。
- 数据保存：增加本地 JSON 备份导出和备份恢复，缓解换电脑、换浏览器或清缓存导致的数据丢失问题。
- 登录控制：演示登录改为哈希校验，并加入 30 分钟会话超时。
- 导入模板：增加标准学生库模板和待核对表模板下载。
- 异常处理：核对结果支持标记处理、填写处理意见、记录处理人和处理时间。
- 敏感导出：完整导出会二次确认，取消则自动脱敏导出。
- 核验规则：可勾选参与自动比对的字段。
- 导入错误：导入后可查看错误来源、Excel 行号、字段、当前值和错误说明。
- CDN 依赖：Excel 解析库不可用时，结果导出仍可降级为 CSV；正式离线部署建议将 SheetJS 改为本地文件引用。
- 备份与恢复：备份文件包含标准库、待核对表、核对结果、导入错误、设置和本地操作日志。

## 注意事项

- 登录功能为前端本地演示登录，用于项目展示和页面访问控制示意，不等同于服务端安全认证。
- 当前项目仍是浏览器本地版，不具备真正的服务端数据库、多人协同、权限分级和集中审计能力；正式处理真实资助数据时建议部署后端服务。
- 真实学生信息、身份证号和银行卡号涉及敏感隐私，请勿上传到公开仓库或公开演示环境。
- 性别建议填写“男”或“女”，其他内容会在核验中标记为格式异常。
- 年龄可为空；如果填写，必须是合理数字。
- 联系方式建议填写 11 位中国大陆手机号。
- 身份证号支持 15 位或 18 位格式，18 位身份证号会校验末位校验码。
- 银行卡号需填写 12-30 位数字，导入时会自动去除空格和横线。
- 资助金额可为空；如果填写，必须是非负数字。
- 手动新增的数据默认进入标准学生库。
- 重置数据会覆盖当前本地标准库、待核对表和核对结果。
